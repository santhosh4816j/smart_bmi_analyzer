# Smart BMI Analyzer by BeachWeather

A premium, fully offline health companion built with Flutter. Calculates BMI
and gives you daily calorie, protein, and water targets, an offline
nutrition/meal database, weight tracking with charts, and local reminders —
no internet connection required after install.

---

## Requirements

- [Flutter SDK](https://docs.flutter.dev/get-started/install) (latest stable, 3.22+)
- Android Studio (Giraffe or newer) with the Flutter & Dart plugins installed
- Android SDK 34 (installed automatically by Android Studio if missing)
- A device or emulator running Android 6.0 (API 23) or newer

## Opening the project

1. Unzip/copy this project folder anywhere on your machine.
2. Open **Android Studio** → **Open** → select the `smart_bmi_analyzer` folder.
3. Android Studio will detect it as a Flutter project and prompt you to run
   `flutter pub get` — accept it (or run it yourself, see below).
4. The first time you open it, Android Studio auto-generates
   `android/local.properties` with the correct SDK paths for your machine.
   If it doesn't, copy `android/local.properties.example` to
   `android/local.properties` and fill in your own `sdk.dir` / `flutter.sdk`
   paths.

## First-time setup from the terminal

```bash
cd smart_bmi_analyzer
flutter pub get

# Generate the Hive model adapters.
# NOTE: hand-written *.g.dart files are already included in lib/core/models/
# so the project compiles immediately without this step. Only run it if you
# add/change a Hive model and need fresh adapters:
flutter pub run build_runner build --delete-conflicting-outputs

flutter run
```

## Building an APK

```bash
flutter build apk --release
```

The signed-with-debug-key release APK will be at
`build/app/outputs/flutter-apk/app-release.apk`. Before publishing to the
Play Store, replace the debug signing config in `android/app/build.gradle`
with your own keystore.

## What's implemented

- **Onboarding & profile setup** — name, age, gender, height, weight, activity level
- **Home dashboard** — greeting, BMI gauge, calorie/protein/water summary cards, daily progress rings
- **BMI calculator** — category, animated gauge, healthy weight range, weight-to-goal, tips
- **Calorie calculator** — Mifflin-St Jeor BMR + maintenance/loss/gain targets
- **Protein calculator** — objective-based range (maintenance / muscle gain / weight loss)
- **Water calculator** — daily target + quick-log buttons feeding the progress tracker
- **Nutrition guide** — the 20 requested foods with macros, searchable and filterable by category, with one-tap logging
- **Meal suggestions** — 7 categories (Weight Loss, Maintenance, Muscle Gain, Vegetarian, Vegan, High Protein, Budget Friendly) with 3 curated meals each
- **Daily goal tracker** — set targets by objective, see live progress rings for calories/protein/water
- **Weight history** — log weight, view a line chart filterable by week/month/year
- **Reminders** — schedule recurring local notifications (water, exercise, weight, meals, protein, custom), fully offline, survives device reboot
- **Settings** — light/dark/system theme, metric/imperial toggle, reset data, privacy note
- **Offline storage** — Hive for structured data, SharedPreferences for simple settings; nothing ever leaves the device

## Simplified vs. the original spec (worth knowing before you ship)

To keep this deliverable buildable and reviewable in one pass, a few "nice to
have" items from the brief were simplified rather than fully built out:

- **Lottie / hero animations**: the app uses Flutter's built-in animation
  APIs (AnimatedContainer, TweenAnimationBuilder, a custom-painted animated
  BMI gauge) instead of bundling actual Lottie JSON files. Drop `.lottie`/
  `.json` files into `assets/animations/` and wire up the `lottie` package
  (already in `pubspec.yaml`) wherever you'd like richer animation.
- **Achievements/streaks and PDF export**: not implemented. The data model
  (`DailyProgress`, `WeightRecord`) already has what you'd need to compute
  streaks; PDF export would use the `pdf` and `printing` packages against
  the same data.
- **Backup/Restore**: Settings currently has Reset Data; a local
  export-to-file/import-from-file flow can be added on top of the existing
  Hive boxes.
- **Hive adapters were hand-written**, not generated by `build_runner`,
  because this build environment has no network access to pub.dev. They
  match what codegen would output — running `build_runner` yourself will
  regenerate them identically (safe to do any time you change a model).
- **Signing**: release builds currently use the debug keystore so
  `flutter build apk --release` works out of the box. Swap in your own
  keystore before publishing.

## Project structure

```
lib/
  core/
    models/       Hive data models (+ hand-written adapters)
    services/      BMI / calorie / protein / water / notification logic
    database/      Hive box setup & seeding
    themes/        Brand color palette + light/dark ThemeData
    utils/         go_router route table
  providers/       Riverpod state (profile, weight, goals, theme, reminders)
  screens/         One folder per feature area
  widgets/         Reusable BMI gauge, dashboard cards, progress rings
  data/            Offline nutrition + meal plan seed data
android/           Standard Flutter Android project (package id
                   com.beachweather.smartbmianalyzer)
assets/icons/      Beach Weather app icon (also baked into mipmap densities)
```
